# Spec：M10 前端界面规范与响应式验收

## 1. 参考基线

当前需求知识库是内部协作工具，不按营销站或展示页处理。界面规范参考 GitHub 上常见 SaaS/admin dashboard 的结构规律：

- shadcn-admin：侧边导航、命令面板、响应式管理后台。
  https://github.com/satnaing/shadcn-admin
- Tremor：面向 dashboard 的 React/Tailwind 组件与数据卡片组织。
  https://github.com/tremorlabs/tremor
- Mosaic Lite：React + Tailwind 的响应式 admin dashboard 模板。
  https://github.com/cruip/tailwind-dashboard-template

这些参考只作为信息密度、响应式栅格和组件职责的基线，不直接引入新的 UI 库。

## 2. 页面定位

- `/dashboard` 是需求协作总览，负责快速进入需求项目、对话和待办关注项。
- `/kb` 是需求项目列表，不承载复杂编辑。
- `/kb/[id]` 是真实工作台，承载录入、查询、拆解、变更和事件回执。
- 页面必须先服务重复使用和扫描效率，避免大面积装饰、营销式 hero 和过度彩色模块。

## 3. 布局规则

- 页面容器使用横向宽屏上限，但不能把大屏浪费成单窄列；宽屏优先采用横向填满加纵向流式布局。
- dashboard 主区优先使用 `grid`，宽屏可拆成主内容和侧栏；卡片列表使用 `auto-fit/minmax` 或明确断点，禁止把不可控文本塞进固定三列。
- 移动端和窄侧栏下所有主要信息必须自然换行；只有按钮短文案、编号、版本号等短 token 可以使用 `whitespace-nowrap`。
- 任何卡片、tile、列表项都必须有 `min-w-0`，长标题用 `truncate` 或 `line-clamp`，正文用 `break-words` 或正常换行。
- 不做卡片套卡片；页面分区用 full-width section 或普通 grid，卡片只用于可重复项目、状态项和工具面板。

## 4. 指标与身份

- KPI/metric tile 只允许放数字、短代码或 2-4 个汉字短状态。
- 禁止把 `产品/开发/测试` 这类长串或复合语义放进 KPI value。
- 角色、权限、业务身份必须用独立角色矩阵或角色行承载，每个角色包含名称、职责摘要和可换行说明。
- 高亮色只用于当前模块、当前身份、关键动作或明确状态；普通统计、选中文档和处理状态保持中性色。

## 5. 文案与密度

- dashboard 文案默认中文，面向产品、开发、测试三类业务用户。
- 标题、说明、按钮、状态标签不能重叠，不能依赖浏览器缩放才能看清。
- 紧凑面板内使用小标题和短句；长解释放到正文区或详情页，不塞进右侧窄栏。
- 输入框 placeholder 必须是空态提示，不得预填具体业务样例，除非用户主动选择示例。

## 6. 响应式验收

每次改 dashboard、`/kb` 或 `/kb/[id]` 后至少按以下宽度人工或截图检查：

- 360px：移动端，所有卡片纵向排列，按钮可换行，不出现横向滚动。
- 768px：平板，双列或单列自适应，长中文不溢出。
- 1440px：桌面，页面横向空间被主内容、侧栏和栅格合理利用，不出现大块无意义空白。
- 1920px：大屏，主容器仍保持可读宽度，但重点内容不能只挤在左侧窄区。

## 7. 当前整改项

- `/dashboard` 的指标格移除 `产品/开发/测试` 长串 value。
- 产品、开发、测试改成独立业务身份矩阵。
- 右侧关注栏和统计栏使用可换行、可自适应的 grid。
- 修正加载骨架缩进和窄屏栅格，避免视觉错位。
